# SOP: Prospect Data Entry and Review

## Purpose

This SOP explains how prospect research data should be entered and reviewed to improve reporting quality and reduce manual cleanup.

## Required Fields

Each prospect record should include:

1. Donor ID
2. Assigned fundraiser
3. Giving capacity
4. Research status
5. Last research date
6. Notes

## Data Entry Steps

1. Confirm that the donor already exists in the donor table.
2. Create or update the prospect record.
3. Enter the assigned fundraiser.
4. Enter giving capacity if available.
5. Update research status.
6. Add research notes.
7. Run the prospect data quality report.
8. Correct any missing or incomplete fields.

## Quality Checks

The data quality report flags:

- Missing donor email
- Missing assigned fundraiser
- Missing giving capacity
- Missing last research date

## Expected Outcome

This process improves data usability for fundraisers and reduces time spent manually checking incomplete records.
